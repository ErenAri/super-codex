# v2.0.0-beta.1 Release Notes (Draft)

Date: 2026-03-06
Channel: `next`

## Highlights

- Added framework kernel/profile primitives and core profile exports.
- Added trust/safety-gate diagnostics integration (`doctor`, `verify --safety-gates`).
- Added core-loop-first guidance in `start` and `guide`.
- Added structured quick-action contract across major commands:
  - `best_next_command`
  - `next_commands`
  - `quick_actions`
- Added onboarding wizard path (`start --wizard`) with non-interactive fallback.
- Added `doctor --explain` with actionable fix plans and rollback hints.
- Added core-mode policy metadata and enforcement:
  - core policy bundle metadata on `fast`, `safe`, `deep`, `deep-research`
  - `safe` mode now gates write-capable workflows behind `--dry-run --explain`
- Added core onboarding agent recommendations in `start` and `guide`.
- Added deterministic golden fixture expansion:
  - `start --wizard --json`
  - `doctor --explain --json`
  - `verify --safety-gates --json`
- Added Windows test-cleanup hardening with retry/backoff helper.
- Added CI benchmark smoke gate for PR on-demand execution with threshold checks + artifact upload.
- Added `PROJECT_INDEX.md` and `docs/UPGRADE.md` for framework-grade onboarding.
- Added project template presets (`api-service`, `web-app`, `library`, `monorepo`) with additive init, JSON/list output, and optional lock refresh for team version control.
- Added release-train automation workflow with weekly canary readiness and dispatch-driven canary/stable lanes.
- Added structured changelog fragment pipeline and generated release notes workflow.

## Behavior Changes

- `run implement` (and other write-capable run workflows) in `safe` mode now require:
  - `--dry-run`
  - `--explain`
- Verification and diagnostics outputs now include shared quick-action fields.
